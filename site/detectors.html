Running a subset

There is no per-id flag. Point --detectors at a directory holding only the TOMLs you want (or disable individual detectors in .keyhog.toml with [detector.<id>] enabled = false):

$ mkdir my-detectors
$ cp detectors/aws-*.toml detectors/stripe-secret-key.toml my-detectors/
$ keyhog scan . --detectors my-detectors/

Or list every detector that matches a filter:

$ keyhog detectors --search aws --verbose
$ keyhog detectors --severity critical --service stripe

Adding a new detector

Detectors are data, not code. Drop a TOML file under detectors/ and rebuild — no Rust changes needed.

# detectors/acme-api-key.toml

[detector]
id        = "acme-api-key"
name      = "Acme Cloud API key"
service   = "acme"
severity  = "critical"
keywords  = ["acme_", "ACME_API_KEY"]

[[detector.patterns]]
regex       = '''(?i)(?:ACME_API_KEY|acme_api_key)\s*[=:]\s*["']?(acme_[a-zA-Z0-9]{40})["']?'''
description = "Acme API key in env var assignment"

[detector.verify]
endpoint    = "https://api.acme.example/v1/me"
header_auth = "Bearer {credential}"
expect_ok   = [200]

See CONTRIBUTING.md for the full schema (companions, checksums, multiline patterns, allowlist overrides, custom severity-by-context rules).
