Why keyhog

Hardware-routed

Auto-detects your hardware on startup and picks the fastest backend: GPU zero-copy (NVMe → GPU DMA), SIMD regex (Hyperscan), or AC + regex on CPU. No flags.

891 service detectors

Service-specific patterns with checksum validation (GitHub CRC32, npm, Slack, PyPI), entropy gating, multiline reassembly, decode-through scanning for base64, JWT, k8s YAML.

SARIF + GitHub code scanning

One-line GitHub Actions integration uploads SARIF to GitHub Advanced Security automatically. CWE-798 + OWASP A07:2021 taxa on every finding.

Hardened by default

Disables core dumps and ptrace on every startup. --lockdown adds mlockall, refuses to start with a stale on-disk cache, and blocks outbound verification.

Daemon mode — 105× faster re-scan

Amortize the ~3 s compile across many invocations. keyhog scan --stdin --daemon returns in ~7 ms. Perfect for pre-commit hooks and IDE save handlers.

Built for scale

Two-phase coalesced scan: phase 1 Hyperscan NFA across all files in parallel (95%+ pay zero cost), phase 2 extraction on hits only. Scales linearly to multi-GB monorepos.

Get started

1. Install

cargo install keyhog — works on Linux, macOS (Intel + Apple Silicon), and Windows. No config.

2. Scan

keyhog scan . — full directory in seconds. Filter by severity, format, or git scope.

3. Ship to CI

One-line GitHub Actions or a pre-commit hook. SARIF lands in GitHub code scanning automatically.
