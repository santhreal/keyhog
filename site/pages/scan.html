Scan command

keyhog scan is the workhorse. Same command for filesystem, stdin, git history, Docker images, S3 buckets, and GitHub orgs.

Synopsis

$ keyhog scan [SOURCE...] [OPTIONS]

If no source is given, the current directory is scanned. All sources can be combined in one invocation; findings stream as they fire.

Backend control

keyhog routes automatically. Override only if you have a reason.

$ KEYHOG_BACKEND=gpu  keyhog scan .   # force GPU zero-copy
$ KEYHOG_BACKEND=simd keyhog scan .   # force Hyperscan
$ KEYHOG_BACKEND=cpu  keyhog scan .   # force CPU fallback
$ KEYHOG_THREADS=8    keyhog scan .   # pin rayon worker count

Inspect the routing matrix at any time:

$ keyhog backend
$ keyhog backend --probe-bytes $((300 * 1024 * 1024))

Verification

$ keyhog scan . --verify              # live-verify all eligible findings
$ keyhog scan . --verify --severity high # only verify high+critical
$ keyhog scan . --verify --verify-timeout 5s

Each verifiable detector ships a verifier (GitHub PATs hit GET /user, AWS keys hit STS GetCallerIdentity, Slack hits auth.test, etc.). Read-only; never writes; refused under --lockdown.

Exit codes

CodeMeaning
0No findings above the severity floor
1One or more findings above the severity floor
2Error: bad path, unreadable file, non-UTF-8 filename, unsupported flag, bad config

This contract is enforced — nonexistent paths and unreadable files exit 2, never silently exit 0. CI gates that rely on $? == 0 won't be misled into reporting clean scans on broken inputs.
