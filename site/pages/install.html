Installation

keyhog is a single Rust binary. Install it with the one-line installer, download a prebuilt release, pull the Docker image, or build from source. Works on Linux, macOS (Intel + Apple Silicon), and Windows with zero configuration. Crates.io publishing lands with the next tagged release; cargo install keyhog doesn't work yet.

One-liner (recommended)

The simplest path. Includes SIMD, ML, entropy, decode-through, and multiline reassembly.

$ curl -fsSL https://raw.githubusercontent.com/santhsecurity/keyhog/main/install.sh | sh

Drops a binary in ~/.local/bin/keyhog. The installer detects your OS, CPU, and GPU before downloading. On Ubuntu/Debian you'll also need the Hyperscan runtime: sudo apt-get install -y libhyperscan5 (the installer prints this hint if the binary fails to start).

Windows

$ iwr https://raw.githubusercontent.com/santhsecurity/keyhog/main/install.ps1 -useb | iex

Drops keyhog.exe in %LOCALAPPDATA%\keyhog\bin and adds it to your PATH.

GPU selection

The default Linux build dispatches via WGPU on any compatible adapter and runs SIMD on the CPU path. If you have the full CUDA toolkit installed (nvcc on PATH or $CUDA_HOME set), pass --variant=cuda to install.sh for the native-CUDA build. See Performance →. An explicit CUDA request fails closed if the CUDA release asset is missing; only auto-selected CUDA hosts may fall back to the WGPU + SIMD asset.

Minimal source build (no SIMD, no ML)

$ git clone https://github.com/santhsecurity/keyhog && cd keyhog
$ cargo build --release -p keyhog --no-default-features --features portable

Drops Hyperscan, ML scoring, and decode-through. ~3 MB smaller binary, ~10× slower per pattern, no entropy/ML gating. Use this only for embedded or locked-down build environments where pulling Hyperscan is a non-starter.

Prebuilt binaries

Every tagged release attaches platform binaries (real names from the releases page):

  • keyhog-linux-x86_64 (WGPU + SIMD default)
  • keyhog-linux-x86_64-cuda (native CUDA dispatch)
  • keyhog-macos-aarch64 (Apple Silicon)
  • keyhog-macos-x86_64 (Intel Mac)
  • keyhog-windows-x86_64.exe
$ curl -L https://github.com/santhsecurity/keyhog/releases/latest/download/keyhog-linux-x86_64 \
    -o keyhog && chmod +x keyhog
$ sudo install keyhog /usr/local/bin/

Docker

$ docker run --rm -v "$PWD:/src" santhsecurity/keyhog scan /src

The image is built from the Dockerfile at the repo root. It bundles the full SIMD + ML feature set, no GPU. ~110 MB compressed.

From source

$ git clone https://github.com/santhsecurity/keyhog.git
$ cd keyhog
$ cargo install --path crates/cli

Local development: cargo run -p keyhog -- scan . from the repo root runs the unmodified binary against your tree.

Verify the install

$ keyhog --version
keyhog v0.5.40

$ keyhog backend
keyhog v0.5.40 | 16 cores | SIMD: AVX-512 | Hyperscan | 922 detectors
  autoroute calibration: fresh
    small-tree/default-detectors → simd-regex
    large-file/default-detectors → gpu-region-presence

Shell completions

# bash
$ keyhog completion bash > /etc/bash_completion.d/keyhog

# zsh
$ keyhog completion zsh > "${fpath[1]}/_keyhog"

# fish
$ keyhog completion fish > ~/.config/fish/completions/keyhog.fish

Upgrading

$ curl -fsSL https://raw.githubusercontent.com/santhsecurity/keyhog/main/install.sh | sh

Or download a fresh release tarball. Baselines, ignore files, and the incremental cache are forward-compatible across patch releases; major version bumps (0.X) call out any migration in the CHANGELOG.
