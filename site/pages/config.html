Configuration

Most users never need a config file — every flag has a sane default. When your team standardizes on a set of defaults (severity floor, exclude patterns, output format), drop a .keyhog.toml at the repo root and commit it.

Precedence

Highest precedence wins (later overrides earlier):

  1. Compiled-in defaults.
  2. System config: /etc/keyhog/config.toml.
  3. User config: ~/.config/keyhog/config.toml.
  4. Repo config: .keyhog.toml walking up from the scanned path.
  5. Environment variables (KEYHOG_*).
  6. CLI flags.

.keyhog.toml — full reference

# Scan defaults
[scan]
severity        = "high"          # info | low | medium | high | critical
min_confidence  = 0.5
max_file_size   = "10M"          # K / M / G suffixes
respect_gitignore = true
exclude = [
  "**/test/fixtures/**",
  "**/*.snap",
  "vendor/",
]
include = []                  # empty = include everything; otherwise allowlist

# Output defaults
[output]
format     = "text"             # text | json | jsonl | sarif
color      = "auto"             # auto | always | never
show_secrets = false
quiet      = false

# Backend routing — usually leave as auto
[backend]
choice  = "auto"                # auto | gpu | simd | cpu | mega-scan
threads = 0                   # 0 = auto-detect (1 per core)

# Detector overrides — disable or down-tune specific detectors
[detector.aws-access-key]
enabled   = true
severity  = "critical"          # override the shipped severity
min_confidence = 0.6          # per-detector confidence floor

[detector.generic-api-key]
enabled = false               # turn off a noisy detector globally

# Allowlists — repo-wide governance metadata
[allowlist]
file      = ".keyhogignore"
require_reason       = true   # reject ignore entries without a reason
require_approved_by  = true
max_expires_days     = 180    # reject expires > 180 days out

# Verification
[verify]
enabled         = false       # --verify must be passed explicitly
timeout_secs    = 5
parallel        = 8
allowed_services = ["github", "aws", "stripe"]   # empty = all

# Incremental cache
[incremental]
enabled = true
cache_path = "~/.cache/keyhog/merkle.idx"

# Daemon
[daemon]
socket_path = "$XDG_RUNTIME_DIR/keyhog.sock"
auto_connect = true           # --daemon attempted by default if socket exists

# Lockdown enforcement — fail closed if the env can't honor lockdown
[lockdown]
require = false               # true = refuse to run without --lockdown

Environment variables

VariableEffect
KEYHOG_BACKENDForce a backend (gpu|simd|cpu|mega-scan)
KEYHOG_THREADSPin rayon worker count
KEYHOG_CONFIGOverride config file search
KEYHOG_CACHE_DIROverride ~/.cache/keyhog/
KEYHOG_NO_COLOR / NO_COLORDisable ANSI output
KEYHOG_DAEMON_SOCKETOverride daemon socket path
KEYHOG_LOGenv_logger filter (trace|debug|info|warn|error)
KEYHOG_LOCKDOWN_REQUIRERefuse to run without --lockdown (same as [lockdown] require)

Verify your config

$ keyhog config print
[scan]
severity = "high"                  ← .keyhog.toml:2
min_confidence = 0.5               ← .keyhog.toml:3
max_file_size = "10M"              ← compiled default
[output]
format = "sarif"                   ← env KEYHOG_FORMAT
...

Each merged value is annotated with where it came from — perfect for "why is the severity floor wrong?" debugging.

Project conventions

  • Commit .keyhog.toml and .keyhogignore at the repo root.
  • Commit .keyhog-baseline.json if you're using baselines.
  • Add .cache/keyhog/ and keyhog.sarif to .gitignore.
  • Don't commit per-user output (--output targets).
