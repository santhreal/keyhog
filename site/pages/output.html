Output formats

Pick text for humans, JSON for one-shot CI parsing, JSONL for streaming pipelines, SARIF for GitHub / GitLab security dashboards.

text (default)

Human-readable, colorized when stdout is a TTY. Findings are severity-sorted, credentials redacted (first 8 + last 4 chars).

.env.production:3
  stripe-secret-key  sk_live_4eC3…W7yz  [critical · 0.99]
  rotate at: https://dashboard.stripe.com/apikeys

terraform/main.tf:42
  aws-access-key  AKIAIOSFODNN7EXAMPLE  [high · 0.96]
  companion: AWS_SECRET_ACCESS_KEY at .env.production:5

2 findings · 0.4 s · 1,247 files scanned

Add --show-secrets to print full credentials (refused under --lockdown). Add --no-color for plain ASCII.

json

Single JSON document — array of findings + run metadata. Use for one-shot CI parsing where you want to load everything into memory.

$ keyhog scan . --format json -o keyhog.json
{
  "version": "0.5.16",
  "scan_id": "01HX7…",
  "started_at": "2026-05-24T16:31:09Z",
  "backend": "simd-regex",
  "files_scanned": 1247,
  "duration_ms": 412,
  "findings": [
    {
      "detector": "stripe-secret-key",
      "severity": "critical",
      "confidence": 0.99,
      "path": ".env.production",
      "line": 3,
      "col": 19,
      "byte_start": 87,
      "byte_end": 134,
      "redacted": "sk_live_4eC3…W7yz",
      "sha256": "9d6060e2…782a8a676a",
      "context": "STRIPE_KEY=sk_live_…",
      "verified": null,
      "rotation_url": "https://dashboard.stripe.com/apikeys",
      "cwe": "CWE-798",
      "owasp": "A07:2021"
    }
  ]
}

Credential values are never emitted in JSON output — only redacted form + a SHA-256 hash you can use to dedupe against your own allowlists. Use --show-secrets if you specifically need plaintext.

jsonl (streaming)

One finding per line, emitted as discovered. Use for pipelines that process findings as they arrive (Splunk, Elasticsearch, Loki, custom dashboards). The first line is the run header; subsequent lines are findings; the final line is the run summary.

$ keyhog scan . --format jsonl --stream | jq '.detector' | sort | uniq -c
{"kind":"header","version":"0.5.16","backend":"simd-regex"}
{"kind":"finding","detector":"stripe-secret-key",…}
{"kind":"finding","detector":"aws-access-key",…}
{"kind":"summary","findings":2,"files_scanned":1247,"duration_ms":412}

sarif

SARIF 2.1.0 output with CWE-798 and OWASP A07:2021 taxa attached to every result. Drops straight into GitHub Advanced Security, GitLab Security Dashboard, Azure DevOps, or any SARIF-aware viewer.

$ keyhog scan . --format sarif -o keyhog.sarif

Each finding is a SARIF result object with:

  • ruleId = detector id (e.g. stripe-secret-key)
  • level = error | warning | note derived from severity
  • locations[] with byte offsets + line/column
  • partialFingerprints.primaryLocationLineHash for GitHub dedup
  • taxa[] with CWE-798 and OWASP A07:2021 references
  • properties.confidence, properties.verified

The GitHub Action uploads SARIF automatically when format: sarif. See CI / SARIF → for the full wiring.

Redaction policy

OutputDefaultWith --show-secrets
textfirst 8 + last 4 charsfull plaintext
json / jsonlredacted + sha256plaintext in credential field
sarifredacted only; sha256 in fingerprintsplaintext in message.text

--lockdown refuses --show-secrets in every format, including text. The redacted form is always deterministic so you can diff scans across commits.

JSON schema

The JSON schema for both --format json and the streaming --format jsonl kind is published at crates/core/schemas/finding.schema.json. It versions independently of the CLI — patch releases never break the schema; minor releases add fields with null defaults.
