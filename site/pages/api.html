Library API

keyhog ships as a CLI, but every layer underneath is a library you can embed. Three crates cover 99% of integration needs.

The crates

CrateWhat it gives you
keyhog-coreDetector loading, finding types, severity, reporting (text/JSON/SARIF), allowlist matching. No scanning.
keyhog-scannerCompiledScanner: compile detectors → scan a Chunk → get Vec<Finding>. Hardware routing, decode-through, entropy, ML.
keyhog-sourcesFile system walker, git source (staged/diff/history), stdin, Docker, S3, GitHub org. Streams Chunks into a scanner.
keyhog-verifierLive verification against vendor APIs (344 detectors with a vendor verifier endpoint).

Minimal example

use keyhog_core::{Chunk, ChunkMetadata};
use keyhog_scanner::CompiledScanner;
use std::path::Path;

fn main() -> anyhow::Result<()> {
    let detectors = keyhog_core::load_detectors(Path::new("detectors"))?;
    let scanner = CompiledScanner::compile(detectors)?;

    let findings = scanner.scan(&Chunk {
        data: "TOKEN=sk_live_EXAMPLE_DOCS_NOT_A_REAL_KEY_z9y8".into(),
        metadata: ChunkMetadata::default(),
    });

    for f in &findings {
        println!("{} at {} [{}]", f.detector_id, f.path.as_deref().unwrap_or("(stdin)"), f.severity);
    }
    Ok(())
}

Add to Cargo.toml using a git path until crates.io publishing lands:

[dependencies]
keyhog-core    = { git = "https://github.com/santhsecurity/keyhog", tag = "v0.5.40" }
keyhog-scanner = { git = "https://github.com/santhsecurity/keyhog", tag = "v0.5.40" }

Embedded detectors (no detectors/ directory)

For library users who don't want to ship a detectors/ directory alongside their binary, the embedded set is always compiled into keyhog-core by build.rs; no feature flag required. Use the fail-closed embedded loader so corrupt bundled TOML stops the scan instead of silently dropping detectors:

let detectors = keyhog_core::load_embedded_detectors_or_fail()?;
let scanner = CompiledScanner::compile(detectors)?;

Custom detectors

Pass your own detector vector into CompiledScanner::compile — the scanner doesn't care where they came from:

use keyhog_core::{DetectorSpec, PatternSpec, Severity};

let mine = vec![DetectorSpec {
    id:       "acme-api-key".into(),
    name:     "Acme API key".into(),
    service:  "acme".into(),
    severity: Severity::Critical,
    keywords: vec!["acme_".into()],
    patterns: vec![PatternSpec {
        regex: r#"(?i)acme_[a-zA-Z0-9]{40}"#.into(),
        ..Default::default()
    }],
    ..Default::default()
}];

let scanner = CompiledScanner::compile(mine)?;

Mix shipped + custom by concatenating before compile:

let mut all = parse_embedded();
all.extend(mine);
let scanner = CompiledScanner::compile(all)?;

Streaming a source

use keyhog_sources::FsSource;
use keyhog_scanner::CompiledScanner;

let scanner = CompiledScanner::compile(parse_embedded())?;
let source = FsSource::new(".")
    .respect_gitignore(true)
    .max_file_size(10 * 1024 * 1024);

for chunk in source.chunks() {
    let findings = scanner.scan(&chunk?);
    for f in findings {
        // …
    }
}

Sources implement Iterator<Item = Result<Chunk>> so they compose with rayon / async-std / tokio however you like.

Parallel scan

use rayon::prelude::*;

let scanner = CompiledScanner::compile(parse_embedded())?;
let chunks: Vec<_> = FsSource::new(".").chunks().collect::<Result<_, _>>()?;

let findings: Vec<_> = chunks.par_iter()
    .flat_map_iter(|c| scanner.scan(c))
    .collect();

The scanner is Send + Sync; share one across rayon workers. The internal Hyperscan scratch space is thread-local, allocated on first use per thread.

Live verification

use keyhog_verifier::Verifier;

let verifier = Verifier::new()?;
for f in &mut findings {
    if let Some(verdict) = verifier.verify(f).await {
        f.verified = Some(verdict);
    }
}

Verifier hits the vendor API read-only; supports timeout, parallelism cap, allowed-services filter. Refused under --lockdown.

Reporting

use keyhog_core::report::{TextReporter, JsonReporter, SarifReporter};
use std::io::stdout;

TextReporter::default().write(&mut stdout(), &findings)?;
JsonReporter::default().write(&mut stdout(), &findings)?;
SarifReporter::default()
    .with_repo_uri("https://github.com/acme/app")
    .write(&mut stdout(), &findings)?;

Stability

The library API follows semver:

  • Patch releases (0.5.x) may add fields with #[non_exhaustive] defaults; never break compile.
  • Minor releases (0.X) may rename or remove items with one minor's notice via #[deprecated].
  • The finding JSON schema is versioned independently — schema_version = "1" on every emitted document; schema changes are explicit, not implicit.

FFI / non-Rust embedding

The keyhog binary is the supported integration path for non-Rust callers. Invoke it with --format jsonl --stream over a subprocess and parse line-delimited findings. The daemon mode + Unix socket protocol is also available; see the protocol spec in the repo under docs/DAEMON_PROTOCOL.md.
