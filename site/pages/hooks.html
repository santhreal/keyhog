Pre-commit hooks

The cheapest place to catch a leaked secret is on the developer's machine, before it ever hits the remote. keyhog ships two equally fast paths.

Bundled installer (simplest)

$ keyhog hook install
installed .git/hooks/pre-commit
  on commit: keyhog scan --fast --git-staged --backend simd

Writes .git/hooks/pre-commit directly. No pre-commit framework needed. The hook runs keyhog scan --fast --git-staged --backend simd over the staged blobs only, exits non-zero on findings, and the commit aborts. If keyhog is missing from PATH, the hook blocks because the scan did not run; install KeyHog, fix PATH, or remove the hook if this repository should not be protected. If a .git/hooks/pre-commit already exists, keyhog hook install refuses to overwrite it; remove the existing file (or run keyhog hook uninstall) and re-install.

$ keyhog hook uninstall                      # remove the keyhog-generated pre-commit hook

For a fast-path scan that reuses the daemon's compiled scanner, start the daemon first (keyhog daemon start) before committing - the in-process scan transparently routes through the running daemon when the socket is reachable.

pre-commit framework

If your team already uses pre-commit, add this to .pre-commit-config.yaml:

repos:
  - repo: https://github.com/santhsecurity/keyhog
    rev: v0.5.41
    hooks:
      - id: keyhog

The hook id is keyhog. By default it runs keyhog scan --git-staged --severity high --daemon. Override with the framework's standard config:

      - id: keyhog
        args: [--severity, critical, --verify]
        stages: [commit, push]

Husky / lefthook

# .husky/pre-commit
#!/usr/bin/env sh
keyhog scan --git-staged --severity high --daemon
# lefthook.yml
pre-commit:
  commands:
    keyhog:
      run: keyhog scan --git-staged --severity high --daemon

Performance matters here

Pre-commit hooks run on every commit; if they take more than a second or two, developers learn to bypass them with --no-verify. Three flags keep keyhog under a second on any reasonable diff:

  • --git-staged — scan only the staged blobs, not the whole tree.
  • --daemon — amortize the ~3 s CompiledScanner::compile cold start across many invocations. Measured 105× speedup (7 ms via daemon vs 740 ms in-process) on a real github_pat sample.
  • --fast — skip ML, entropy, decode-through. Pre-commit is a fast first pass; the full scan runs in CI.

Combine all three for the fastest possible hook:

$ keyhog scan --git-staged --fast --daemon --severity high

Bypass policy

Developers can bypass with git commit --no-verify; that's git, not keyhog. To make bypass impossible:

  • Run keyhog in CI on every PR (see CI / SARIF →). The pre-commit hook is a developer convenience; CI is the gate.
  • Pair with a server-side hook (pre-receive) on your git forge — see DROP_IN_USAGE.md for a Gitea / GitLab / Forgejo example.

What the hook actually does

The installed hook is plain shell — you can read and edit it:

$ cat .git/hooks/pre-commit
#!/usr/bin/env sh
# Installed by `keyhog hook install` on 2026-05-24.
# Edit freely; `keyhog hook uninstall` will restore the prior hook.

KEYHOG=${KEYHOG_BIN:-keyhog}
if ! command -v "$KEYHOG" >/dev/null 2>&1; then
  echo "keyhog not on PATH; skipping (install: curl -fsSL .../install.sh | sh)"
  exit 0
fi
exec "$KEYHOG" scan --git-staged --severity high --daemon

If keyhog isn't on PATH the hook prints a hint and passes, so adding the binary later doesn't require reinstalling the hook.
