Quickstart

Scan a directory, filter by severity, output JSON for CI. Three commands cover 80% of usage.

1 — Scan a directory

$ keyhog scan .

keyhog uses the persisted autoroute calibration for the workload, walks the tree (respecting .gitignore), and prints findings as they fire. If calibration is missing or stale, the scan exits with an actionable error instead of guessing. Default output is human-readable text, redacted credentials, severity-sorted.

Exit codes: 0 = no findings, 1 = findings above the severity floor, 2 = error (bad path, unreadable file, unsupported argument).

2 — Filter what fires

$ keyhog scan . --severity high            # medium+ ignored
$ keyhog scan . --min-confidence 0.7       # raise the ML floor
$ keyhog scan . --exclude-paths '**/test/**' # skip test trees
$ keyhog scan . --fast                     # pattern-only scan path

Severity is one of info · low · medium · high · critical. Combine flags freely — they AND together.

3 — JSON / SARIF for CI

$ keyhog scan . --format json -o keyhog.json
$ keyhog scan . --format sarif -o keyhog.sarif

SARIF outputs CWE-798 and OWASP A07:2021 taxa on every finding so it slots straight into GitHub Advanced Security, GitLab Security Dashboard, or any SARIF-aware tool.

Common workflows

Pre-commit hook

Block secrets before they land on disk. keyhog hook install or via the pre-commit framework.

GitHub Actions

One uses: line. Auto-uploads SARIF to code scanning. Release tags fail closed; branch/SHA refs may build from source.

Adopt an existing repo

Snapshot today's findings, then only break the build on new ones. keyhog scan --create-baseline.

Allowlist a known leak

Drop a hash, path glob, or detector id in .keyhogignore. Add reason + expires for governance.

Other useful invocations

# Scan only what's staged for commit (perfect for pre-commit)
$ keyhog scan --git-staged

# Diff against a branch / tag
$ keyhog scan --git-diff main

# Full git history (every commit, every branch — slow)
$ keyhog scan --git-history .

# Pipe through stdin
$ cat config.yaml | keyhog scan --stdin

# Scan a Docker image (unpacks `docker image save`)
$ keyhog scan --docker-image registry.example/app:v1.2.3

# Scan an S3 bucket (bucket name; --s3-prefix to narrow)
$ keyhog scan --s3-bucket logs-prod

# Scan an entire GitHub org
$ keyhog scan --github-org santhsecurity --github-token "$GITHUB_TOKEN"

Live verification

$ keyhog scan . --verify

For findings that keyhog knows how to verify (GitHub PATs, AWS keys, Slack tokens, Stripe keys, OpenAI keys, 341 detectors with a vendor verifier endpoint), keyhog hits the vendor API read-only to confirm the credential is live. Unverified findings are marked with [unverified]; live ones get [verified ✓].

Verification sends credentials over the network. Disabled by default; refuses to run under --lockdown. Pair with --severity high to limit the verification cost.
