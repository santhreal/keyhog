Reproduce the benchmarks

$ git clone https://github.com/santhsecurity/keyhog.git
$ cd keyhog
$ cargo bench --bench scan_throughput
$ cargo bench --bench cold_start
$ cargo bench --bench daemon_latency

The Django / Kubernetes / monorepo numbers come from the benchmark-harness/ directory. Run ./benchmark-harness/run.sh to reproduce; the script vendors the comparison corpora, runs every tool with their best-effort flags, and emits a Markdown table.

Profiling tips

  • KEYHOG_LOG=debug dumps per-phase timing on stderr.
  • keyhog scan . --bench prints per-detector hit counts and time spent in each phase.
  • perf record / cargo flamegraph work cleanly — keyhog is a pure Rust binary with no async runtime, so stack traces are clean.
  • Daemon mode hides the compile cost; if your measured "scan time" is dominated by ~3 s, you're measuring the cold start, not the scan.

What "fastest" actually means

"Fastest secret scanner" is a benchmark trap — Gitleaks scans 86 MB in 0.3 s because it only checks ~50 patterns. keyhog scans the same tree in 0.5 s with 891 patterns, catches 33% more real secrets, and pulls ahead on every dataset above ~200 MB (where the two-phase NFA wins big over linear regex). The right comparison is throughput per pattern at scale, not latency on a tiny tree.
