Performance

Performance claims are generated from committed benchmark artifacts and reproducible harnesses. The website does not carry hand-written leaderboard numbers because stale benchmark tables are worse than no benchmark table.

Benchmark evidence

ArtifactContract
benchmarks/reports/leaderboard.mdGenerated competitor comparison, not edited by hand.
benchmarks/reports/perf.mdGenerated throughput and latency report for the current binary.
benchmarks/results/*.jsonMachine-readable inputs consumed by the source gate and CI bench gate.
scripts/gates/run_all.shRuns the source gates and loud-skips asset gates when benchmark inputs are absent.

Release claims should come from those generated files. If the reports are absent, stale, or missing a competitor, the claim is not made here.

Autoroute contract

Autoroute is a persisted calibration contract, not a hardcoded size table. Installation or keyhog calibrate measures the available backends for workload classes, stores the winning correct backend with hardware and detector-set fingerprints, and normal scans use that record with no per-scan probing.

StateRuntime behavior
Fresh calibration matches host + detector set + workload classUse the persisted backend decision.
Calibration missing, stale, or cannot be readExit with a visible calibration error and the command to repair it.
Backend forced for diagnosticsHonor the explicit override and report the decision inputs.
Backend cannot return equivalent findingsDo not select it for autoroute.

Override with keyhog scan --backend {gpu|simd|cpu|mega-scan} for diagnostic runs. Production routing depends on calibration evidence.

Inspect calibration on your hardware

$ keyhog backend

  cpu: AMD Ryzen 9 7950X · 16c/32t · AVX-512 + AVX2 + SSE4.2
  gpu: NVIDIA RTX 4090 · 24 GB · CUDA + Vulkan
  autoroute calibration: fresh
    small-tree/default-detectors → simd-regex
    large-file/default-detectors → gpu-zero-copy

$ keyhog backend --probe-bytes $((300 * 1024 * 1024))
300 MiB probe class uses persisted calibration → gpu-zero-copy

Reproduce the benchmarks

$ git clone https://github.com/santhsecurity/keyhog.git
$ cd keyhog
$ make -C benchmarks bench
$ make -C benchmarks report

The benchmark harness records machine-readable results under benchmarks/results/ and renders reports under benchmarks/reports/. The prevention gate consumes those files instead of trusting hand-written site numbers.

Profiling tips

  • RUST_LOG=keyhog=debug dumps per-phase routing + timing detail on stderr.
  • keyhog scan --benchmark runs the built-in backend benchmark corpus and prints throughput per backend, then exits (no path argument needed).
  • perf record / cargo flamegraph work cleanly — keyhog is a pure Rust binary with no async runtime, so stack traces are clean.
  • Daemon mode hides the compile cost; if your measured "scan time" is dominated by ~3 s, you're measuring the cold start, not the scan.

What the comparison means

Secret-scanner speed is only meaningful when the compared tools scan the same corpus, use their strongest available path, and return equivalent findings. keyhog's benchmark gate treats recall, false positives, throughput, and autoroute correctness as one contract.
