Scan command

keyhog scan is the workhorse. Same command for filesystem, stdin, git history, Docker images, S3 buckets, and GitHub orgs.

Synopsis

$ keyhog scan [SOURCE...] [OPTIONS]

If no source is given, the current directory is scanned. All sources can be combined in one invocation; findings stream as they fire.

Sources

Flag / formWhat it scans
keyhog scan PATHDirectory or file (respects .gitignore)
--stdinRead bytes from stdin (e.g. cat | keyhog scan --stdin)
--git-stagedOnly files staged for the next commit
--git-diff REFFiles changed since branch / tag / commit
--git-history PATHEvery blob in every commit of every branch
--docker-image REFDocker image layers (pulls if not cached)
--s3-bucket BUCKET + --s3-prefix PFXS3 objects (uses default AWS credentials)
--github-org ORG + --github-token PATEvery accessible repo in the org
--url URL ...Fetch + scan one or more URLs (JS / sourcemap / WASM / text)

Filtering

FlagDefaultEffect
--severity LEVELinfoDrop findings below LEVEL (info|low|medium|high|critical)
--min-confidence F—Drop findings below confidence score F (0.0-1.0)
--detectors DIRdetectorsLoad detector TOMLs from DIR. To run a subset, point this at a directory holding only the detectors you want — there is no per-ID enable/disable flag.
--exclude-paths GLOB ...—Skip paths matching glob (space-separated, repeatable)
--max-file-size BYTES—Skip files above size; summary line at the end
--baseline FILE—Only report findings NOT in the baseline

Speed knobs

FlagEffect
--fastSkip ML, entropy, and decode-through. Pre-commit speed.
--deepMaximum detection depth: multi-level decode, generic+entropy fallback, all companions checked.
--incrementalBLAKE3 Merkle index — unchanged files skip the scanner. 10–100× CI speedup.
--incremental-cache FILEOverride cache path (default ~/.cache/keyhog/merkle.idx)
--daemonConnect to a running keyhog daemon over Unix socket (~105× cold-start savings)

Backend control

keyhog routes through persisted calibration evidence. Override only if you need a diagnostic run.

$ keyhog scan --backend gpu .   # force GPU zero-copy
$ keyhog scan --backend simd .  # force SIMD/Hyperscan path
$ keyhog scan --backend cpu .   # force scalar CPU path
$ keyhog scan --threads 8 .   # pin rayon worker count

Inspect the routing matrix at any time:

$ keyhog backend
$ keyhog backend --probe-bytes $((300 * 1024 * 1024))

Verification

$ keyhog scan . --verify              # live-verify all eligible findings
$ keyhog scan . --verify --severity high # only verify high+critical
$ keyhog scan . --verify --verify-rate 2   # cap verify calls/sec per service

Each verifiable detector ships a verifier (GitHub PATs hit GET /user, AWS keys hit STS GetCallerIdentity, Slack hits auth.test, etc.). Read-only; never writes; refused under --lockdown.

Exit codes

CodeMeaning
0No findings above the severity floor
1One or more findings above the severity floor
2Error: bad path, unreadable file, non-UTF-8 filename, unsupported flag, bad config

This contract is enforced — nonexistent paths and unreadable files exit 2, never silently exit 0. CI gates that rely on $? == 0 won't be misled into reporting clean scans on broken inputs.
