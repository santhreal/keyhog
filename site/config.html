Configuration

Most users never need a config file — every flag has a sane default. When your team standardizes on a set of defaults (severity floor, exclude patterns, output format), drop a .keyhog.toml at the repo root and commit it.

Precedence

Highest precedence wins (later overrides earlier):

  1. Compiled-in defaults.
  2. Repo config: the nearest .keyhog.toml found by walking up from the scanned path (or an explicit --config <path>).
  3. CLI flags.

.keyhog.toml — full reference

# Scan defaults — flat top-level keys are the canonical form. Every key
# mirrors a `keyhog scan` flag; the CLI flag overrides it.
detectors        = "detectors"     # detector TOML directory; --detectors overrides
severity         = "high"          # info | low | medium | high | critical
min_confidence   = 0.5                # 0.0 - 1.0
format           = "text"          # text | json | jsonl | sarif
fast             = false              # pattern-only, fastest
deep             = false              # every feature, max recall
threads          = 0                  # omit for auto (one per physical core)
# reader_threads = 2              # omit for scan-pool-derived default
fused_batch      = 32                 # fused filesystem batch size
# fused_depth = 4                 # omit for worker-derived default
dedup            = "credential"    # credential | file | none
show_secrets     = false
exclude_paths    = ["**/test/fixtures/**", "vendor/"]
max_file_size    = "100M"          # skip files larger than this
regex_dfa_limit  = "1M"            # per-regex DFA cache ceiling
no_decode        = false
no_entropy       = false
no_ml            = false
no_unicode_norm  = false

# Verification (flat keys)
verify  = false               # same as --verify; must be opted into
timeout = 5                   # per-verify-call timeout, seconds
rate    = 5                   # max verify calls/sec per service

# Equivalent nested form. The [scan] table mirrors a subset of the flat
# keys; set either shape (the flat key wins if both are present).
[scan]
severity        = "high"
min_confidence  = 0.5
format          = "text"
exclude         = ["vendor/"]
threads         = 0
# reader_threads = 2
fused_batch     = 32
# fused_depth = 4
dedup           = "credential"

# Per-detector overrides, keyed by the id from `keyhog detectors`
# (the fast-path ids are prefixed `hot-`, e.g. hot-aws_key).
[detector.generic-api-key]
enabled        = false        # turn off a noisy detector

[detector.aws-access-key]
min_confidence = 0.6          # per-detector confidence floor

# Lockdown: fail closed if the repo mandates hardening
[lockdown]
require = true                # refuse to run unless --lockdown is passed

Suppress specific findings (rather than whole detectors) with a .keyhogignore file, by credential hash, path glob, or detector: id - with optional inline ; reason="..."; expires=YYYY-MM-DD; approved_by="..." metadata (expired entries are dropped automatically). See the Allowlists page.

Verify your config loaded

There is no separate config-dump command. To confirm which .keyhog.toml was picked up and that it parsed, run a scan with debug logging:

$ RUST_LOG=keyhog=debug keyhog scan . 2>&1 | grep keyhog.toml
loaded .keyhog.toml path=/repo/.keyhog.toml

A malformed config prints a WARNING: Failed to parse .keyhog.toml to stderr and is skipped rather than applied silently.

Project conventions

  • Commit .keyhog.toml and .keyhogignore at the repo root.
  • Commit .keyhog-baseline.json if you're using baselines.
  • Add .cache/keyhog/ and keyhog.sarif to .gitignore.
  • Don't commit per-user output (--output targets).
