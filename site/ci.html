CI / SARIF

One-line GitHub Actions integration. Native SARIF for GitHub Advanced Security. Drop-in YAML for every major CI. Verified release assets with branch/SHA source builds.

GitHub Actions

name: secrets
on: [pull_request, push]
jobs:
  keyhog:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      security-events: write   # for SARIF upload
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0      # needed for --git-history
      - uses: santhsecurity/keyhog/.github/actions/keyhog@v0.5.40
        with:
          path: .
          severity: high        # info | low | medium | high | critical
          format: sarif          # text | json | jsonl | sarif
          verify: 'false'         # live-verify findings against vendor APIs

The action auto-downloads a prebuilt linux-x86_64 / macos-aarch64 binary attached to the matching release tag. Release tags and explicit version inputs require a matching asset plus checksum and fail closed if the release asset is missing or unverifiable. Branch/SHA action refs may build from source with Cargo. SARIF is auto-uploaded to GitHub code scanning when format: sarif.

Block PRs on new findings only

      - uses: santhsecurity/keyhog/.github/actions/keyhog@v0.5.40
        with:
          path: .
          baseline: .keyhog-baseline.json
          severity: high
          format: sarif

The baseline file lives in the repo; the action exits 1 only when new findings (not in the baseline) fire. See Baselines →.

Scan a Docker image after build

      - name: build image
        run: docker build -t app:${{ github.sha }} .
      - uses: santhsecurity/keyhog/.github/actions/keyhog@v0.5.40
        with:
          image: app:${{ github.sha }}
          format: sarif

GitLab CI

keyhog:
  image: rust:1.89-slim
  stage: test
  before_script:
    - apt-get update && apt-get install -y --no-install-recommends ca-certificates curl
    - curl -L https://github.com/santhsecurity/keyhog/releases/latest/download/keyhog-linux-x86_64 -o keyhog && chmod +x keyhog
    - install keyhog /usr/local/bin/
  script:
    - keyhog scan . --format sarif -o keyhog.sarif --severity high
  artifacts:
    reports:
      sast: keyhog.sarif
    when: always
    paths: [keyhog.sarif]

SARIF lands in GitLab's Security Dashboard via the artifacts.reports.sast contract.

Why SARIF is the right CI format

  • One schema → drops into GitHub Advanced Security, GitLab Security Dashboard, Azure DevOps SARIF viewer, and any IDE SARIF plugin.
  • Per-result partialFingerprints.primaryLocationLineHash means GitHub correctly dedupes findings across PRs (same secret, same line, even if the line moves).
  • CWE-798 + OWASP A07:2021 taxa on every result, so dashboards can group / filter by industry-standard taxonomy without per-tool configuration.
  • Native severity → SARIF level mapping (critical/high → error, medium → warning, low/info → note) so PR review surfaces look right.

Optional: live verification in CI

      - uses: santhsecurity/keyhog/.github/actions/keyhog@v0.5.40
        with:
          path: .
          verify: 'true'
          severity: high   # only verify high+critical to limit API calls
        env:
          # Provide read-only creds for the services you want verified.
          # keyhog never uses these creds for anything but verification.
          GITHUB_VERIFY_TOKEN: ${{ secrets.KEYHOG_GITHUB_READ_TOKEN }}
Verification sends credentials over the network. Only enable it on PRs from trusted authors. The default GitHub Action sets verify: 'false'; that is the right default for forked-PR workflows.

Speed in CI

Two flags pull most of the CI time out:

  • --incremental persists a BLAKE3 Merkle index; unchanged files skip the scanner entirely on subsequent runs. 10–100× speedup on PRs that touch a few files.
  • --git-diff $BASE_REF scans only changed files vs the PR base, instead of the whole tree.
      - uses: santhsecurity/keyhog/.github/actions/keyhog@v0.5.40
        with:
          git-diff: ${{ github.event.pull_request.base.sha }}
          incremental: 'true'

Cache the incremental DB across runs with actions/cache keyed on ~/.cache/keyhog/merkle.idx.
